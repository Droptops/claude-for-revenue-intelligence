# healthcare-patient-acquisition

Thin overlay for a healthcare patient-acquisition motion.

What changed vs. the base harness:

- Replaces enterprise account slots with referral, episode, and outcome slots.
- Uses practice-growth and operations shaped plugin defaults.
- Keeps health-specific thresholds as fork-local theory constants.

This fork is a stub. It does not implement full agents and does not include
patient data.
